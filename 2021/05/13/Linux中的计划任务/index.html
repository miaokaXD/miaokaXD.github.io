Linux中的计划任务

本文最后更新于：2022年11月3日 晚上

相关知识

在Linux系统中，常见的能实现定时任务的软件有At 、Cron 和 Anacron。

其中，各个服务有其独特的点：

  • At：适合仅执行一次就结束的调度命令，需要在后台启动守护进程atd；

  • Cron：适合脚本、单句命令的调度，需要在后台启动守护进程Crond，并通过Crondtab命令实现；

  • Anacron：适合实现以天为周期，在关机状态下未执行的任务可在开机后继续执行。

有些计划任务是只需要执行一次就能完成的，这时候就需要使用到At服务啦；本文将着重介绍At服务。

了解At

一般情况下At服务默认安装并开启。

如果系统没有安装，可以输入以下命令安装At服务yum install -y at。

常见命令

参数说明

参数 说明
-v 显示版本信息
-q [a-z] 指定新任务的队列名（从a-z，执行优先级依次递减），默认为a队列
-f [file] 指定保存的计划任务文件的路径，不写则为默认目录
-l 显示计划任务列表，等同于 atq 命令
-d 删除指定的待执行的计划任务，等同于 atrm 命令
-m 计划任务完成后自动向用户发送邮件
-c [job] 显示计划任务的详细信息，参数后面接任务号
-t 以定时间参数执行计划任务

提示：使用 -f 参数的时候，最好指定的是绝对路径，以避免不必要的麻烦；同时At计划任务的默认目录为 /var/spool/at/ 中。

同时，At命令支持多种时间参数（格式）

格式 解释 示例
HH:MM (时:分,24小时制) 在当日的HH:MM内执行；如果当日未执行，则往后顺延一日 04:00 当日凌晨4点
HH:MM [am or pm] (时:分,12小时制) 在当日的 HH:MM内（12小时制）执行 如果当日未执行，则往后顺延一日 04:00pm 当日下午4点
YYYY-MM-DD (年-月-日) 强制规定在某年某月某日执行该任务 2021-06-16 2021年6月16日
HH:MM YYYY-MM-DD (时:分,24小时制 年-月-日) 强制规定在某年某月某日某时某分执行该任务 08:00 2021-06-16 2021年6月16日上午8点
HH:MM [am or pm] [Month] [Date] (时:分,12小时制 月 日) 强制规定在某月某日某时某分（12小时制）执行该任务 04:00pm 2021-06-16 2021年6月16日下午4点
Now + X [minutes or hours or days or weeks] (现在时间往后X分钟、小时、天、周 ) 在现在时间往后X分钟、小时、天、周 后执行该任务 Now + 5 minutes 现在时间5分钟后

黑白名单

为了提高安全性，可以通过修改At的名单文件，允许或限制特定用户使用At服务。涉及At的名单文件有两个，详情请见下表。

类别 文件绝对路径 优先级 作用
白名单 /etc/at.allow 高 只有在该文件内写入的用户才能使用At服务。
黑名单 /etc/at.deny 其次 在该文件内写入的用户，将无法使用At服务。
